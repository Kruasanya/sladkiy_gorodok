# Итоговая таблица `sales_by_counterparties`

Файлы:

- `sales_by_counterparties.csv`
- `sales_by_counterparties.xlsx`
- `sales_by_counterparties.parquet`

Источник: Excel-отчеты из папки `Данные/Продажи по контрагентам`.

Гранулярность таблицы: **одна строка = одна строка номенклатуры внутри документа реализации или корректировки**.

## Как собирается таблица

Исходный Excel устроен иерархически:

1. контрагент;
2. договор;
3. документ продажи или корректировки;
4. номенклатура;
5. пары дневных колонок `Количество` и `Сумма`.

Пайплайн делает длинную таблицу:

- контрагент, договор и документ протягиваются на каждую строку номенклатуры;
- дата продажи `sale_date` берется из дневной колонки;
- если дата документа продажи распарсилась и отличается от даты колонки, в `sale_date` записывается дата документа;
- количество записывается в `quantity`;
- сумма записывается в `amount`;
- поля `source_*` показывают происхождение строки в исходном Excel.

## Поля таблицы

| Поле | Тип | Описание |
|---|---|---|
| `counterparty_raw` | string | Полный контрагент/точка как в Excel. |
| `legal_entity` | string | Юрлицо или группа, извлеченная из контрагента. |
| `brand` | string | Бренд сети, например `Светофор` или `Маяк`. |
| `store_location_raw` | string | Адресная часть контрагента. |
| `city_or_area` | string | Город/населенный пункт, если удалось извлечь. |
| `contract_raw` | string | Договор как в Excel. |
| `contract_number` | string | Номер договора без даты. |
| `contract_date` | date | Дата договора. |
| `sales_doc_raw` | string | Документ продажи/корректировки как в Excel. |
| `sales_doc_type` | string | Тип документа: `Реализация` или `Корректировка реализации`. |
| `sales_doc_number` | string | Номер документа продажи/корректировки. |
| `sales_doc_date` | date | Дата документа продажи/корректировки. |
| `sale_date` | date | Итоговая дата продажи для аналитики. |
| `nomenclature` | string | Номенклатура/товар. |
| `quantity` | float | Количество по строке номенклатуры. |
| `amount` | float | Сумма по строке номенклатуры. |
| `period_label` | string | Период из имени исходного файла. |
| `period_start` | date | Начало периода, извлеченное из имени файла. |
| `period_end` | date | Конец периода, извлеченный из имени файла. |
| `source_file` | string | Имя исходного Excel-файла. |
| `source_file_path` | string | Относительный путь к исходному файлу внутри проекта. |
| `source_file_hash` | string | SHA-256 хэш исходного файла для инкрементальной обработки. |
| `source_sheet` | string | Имя листа-источника. |
| `source_row_number` | integer | Номер строки номенклатуры в исходном Excel, начиная с 1. |
| `source_quantity_column` | string | Дата колонки Excel, где было найдено количество. |
| `source_amount_column` | string | Дата колонки Excel, где была найдена сумма. |

## Связь с таблицей оплат

С таблицей `sales_by_payment_counterparties` эту таблицу лучше связывать не по документам, а по измерениям:

- `counterparty_raw`
- `contract_number`
- `contract_date`
- месяц от `sale_date` / месяц от `payment_date`

Документы продаж (`sales_doc_*`) и документы оплат (`payment_doc_*`) являются разными бизнес-сущностями.
