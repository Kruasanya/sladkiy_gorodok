# Итоговая таблица `sales_by_payment_counterparties`

Файлы:

- `sales_by_payment_counterparties.csv`
- `sales_by_payment_counterparties.xlsx`

Источник: Excel-отчеты из папки `Данные/Продажи по контрагентам (по оплате)`.

Гранулярность таблицы: **одна строка = один платеж / одно поступление на расчетный счет**.

## Как собирается таблица

Исходный Excel имеет иерархическую структуру:

1. строка контрагента;
2. строка договора;
3. одна или несколько строк документов `Поступление на расчетный счет`;
4. даты периода разложены по колонкам.

Пайплайн разворачивает этот отчет в длинный формат:

- контрагент и договор протягиваются вниз на строки платежей;
- сумма берется из ненулевой дневной колонки документа;
- дата сначала берется из дневной колонки;
- если дата в тексте документа распарсилась и отличается от даты колонки, итоговой датой считается дата документа;
- итоговая дата записывается в поле `payment_date`.

## Поля таблицы

| Поле | Тип | Описание |
|---|---|---|
| `source_file` | string | Имя исходного Excel-файла. |
| `source_file_path` | string | Относительный путь к исходному файлу внутри проекта. |
| `source_file_hash` | string | SHA-256 хэш исходного файла. Используется для инкрементальной обработки. |
| `source_sheet` | string | Имя листа-источника. Для этих отчетов обычно `Лист_1`. |
| `source_row_number` | integer | Номер строки документа в исходном Excel, начиная с 1. |
| `source_column` | string | Название колонки Excel, где была найдена сумма, например `10.04.26`. |
| `period_label` | string | Период из имени файла. |
| `period_start` | date | Начало периода, извлеченное из имени файла. |
| `period_end` | date | Конец периода, извлеченный из имени файла. |
| `counterparty_raw` | string | Полное название контрагента/точки как в Excel. |
| `legal_entity` | string | Юрлицо или группа, извлеченная из `counterparty_raw`. |
| `brand` | string | Бренд сети, например `Светофор` или `Маяк`. |
| `store_location_raw` | string | Адресная часть контрагента, если ее удалось отделить. |
| `city_or_area` | string | Город/населенный пункт, если его удалось извлечь простым правилом. |
| `contract_raw` | string | Договор как в Excel. |
| `contract_number` | string | Номер договора без даты. |
| `contract_date` | date | Дата договора. |
| `payment_doc_raw` | string | Документ поступления как в Excel. |
| `payment_doc_number` | string | Номер документа поступления. |
| `payment_date` | date | Итоговая дата платежа для аналитики. |
| `amount` | float | Сумма платежа. |
| `counterparty_period_total` | float | Итог по контрагенту в рамках исходного отчета. |
| `report_total` | float | Общий итог исходного отчета. |

## Типовые запросы

Сумма по дням:

```sql
select payment_date, sum(amount) as amount
from sales_by_payment_counterparties
group by payment_date
order by payment_date;
```

Сумма по юрлицам и брендам:

```sql
select legal_entity, brand, sum(amount) as amount
from sales_by_payment_counterparties
group by legal_entity, brand
order by amount desc;
```
