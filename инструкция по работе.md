# Инструкция по работе

## Установка репозитория

### 1. Установить Git

Git нужен, чтобы скачать проект с GitHub и отправлять изменения обратно.

Проверить, установлен ли Git:

```bash
git --version
```

`bash` здесь означает команду в терминале.

На Mac терминал называется **Terminal**.  
На Windows можно использовать **Git Bash** или терминал в VS Code.

Если Git не установлен:

- Mac: после команды `git --version` система предложит установить Command Line Tools.
- Windows: скачать Git с сайта https://git-scm.com/downloads и установить с настройками по умолчанию.

### 2. Установить VS Code

Скачать VS Code: https://code.visualstudio.com/

### 3. Скачать репозиторий

Открыть терминал и перейти в папку, куда нужно скачать проект.

Например, на рабочий стол:

```bash
cd Desktop
```

Скачать проект:

```bash
git clone https://github.com/Kruasanya/sladkiy_gorodok.git
```

После этого появится папка `sladkiy_gorodok`.

### 4. Открыть проект в VS Code

Перейти в папку проекта:

```bash
cd sladkiy_gorodok
```

Открыть проект в VS Code:

```bash
code .
```

Если команда `code .` не работает, открыть VS Code вручную:

```text
File -> Open Folder -> sladkiy_gorodok
```

## Работа с изменениями

### 1. Перед началом работы подтянуть свежую версию

Выполнять внутри папки проекта:

```bash
git pull origin main
```

### 2. Отредактировать файлы

Редактировать файлы в VS Code.

### 3. Сохранить изменения

Есть два варианта: через Codex или вручную через Git.

## Вариант 1. Через Codex

Написать Codex:

```text
/cpd
```

Codex сам проверит изменения, создаст коммит и отправит его на GitHub.

Можно сразу указать текст коммита:

```text
/cpd Добавил новые документы
```

## Вариант 2. Вручную через Git

Проверить изменения:

```bash
git status
```

Добавить изменения в коммит:

```bash
git add --all
```

Создать коммит:

```bash
git commit -m "Короткое описание изменений"
```

Пример:

```bash
git commit -m "Добавил инструкцию по работе"
```

Отправить изменения на GitHub:

```bash
git push origin main
```
