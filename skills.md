# Skills

## `/cpd` - commit, push, done

Команда для Codex: если пользователь пишет `/cpd`, нужно самостоятельно закоммитить и запушить текущие изменения в GitHub.

### Что делать

1. Проверить текущую ветку и состояние репозитория:

```bash
git status --short --branch
git remote -v
```

2. Если изменений нет, сообщить пользователю, что коммитить нечего.

3. Если есть изменения, добавить их в индекс.

По умолчанию не добавлять служебные файлы:

- `.DS_Store`
- файлы, которые начинаются с `~$`

Команда:

```bash
git add --all -- ':!.DS_Store' ':!~$*'
```

Если Git ругается на `~` в pathspec, использовать более явную форму:

```bash
git add --all -- ':!.DS_Store' ':(exclude)~$*'
```

4. Проверить, что попало в коммит:

```bash
git status --short
```

5. Создать коммит.

Если пользователь написал текст после команды, использовать его как сообщение коммита.

Пример:

```text
/cpd Добавил инструкцию по работе
```

Коммит:

```bash
git commit -m "Добавил инструкцию по работе"
```

Если пользователь написал только `/cpd`, самому выбрать короткое сообщение по смыслу изменений.

6. Отправить изменения в текущую ветку.

Сначала узнать ветку:

```bash
git branch --show-current
```

Потом выполнить push:

```bash
git push origin <branch>
```

Например:

```bash
git push origin main
```

7. После push проверить статус:

```bash
git status --short --branch
```

8. Коротко написать пользователю:

- какой коммит создан;
- в какую ветку сделан push;
- какие основные файлы вошли в коммит.

### Если push не прошёл

Если GitHub недоступен или нет прав, сообщить точную ошибку и команду, которую нужно повторить.

Если локальная ветка отстала от GitHub, сначала выполнить:

```bash
git pull --rebase origin <branch>
```

После успешного rebase повторить:

```bash
git push origin <branch>
```
